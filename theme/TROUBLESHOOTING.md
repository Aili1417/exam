# 极简主题故障排除指南

## 常见问题及解决方案

### 问题1: SVIP会员无法使用主题功能

#### 症状
- 已经是SVIP会员
- 主题按钮显示锁定图标🔒
- 点击提示需要升级会员
- 控制台显示"[事件] 已触发 userLoggedIn 事件"但按钮仍锁定

#### 原因
主题管理器初始化时，用户自动登录还未完成，导致：
1. 初始权限检查失败（用户未登录）
2. 事件监听器可能注册太晚，错过了 `userLoggedIn` 事件

#### 解决方案（已修复 v1.2.0）

**自动修复机制**
最新版本实现了四重检查机制，会自动解决此问题：

1. **事件监听**：监听 `userLoggedIn` 事件（最快，<100ms）
2. **立即检查**：注册监听器时立即检查是否已登录
3. **快速轮询**：每0.5秒检查用户状态（检测到后自动停止）
4. **备用轮询**：每1秒持续检查（兜底方案）

**如果仍有问题，尝试以下方案：**

**方案A: 强制刷新页面**
1. 按 `Ctrl + Shift + R`（Windows）或 `Cmd + Shift + R`（Mac）
2. 清除缓存并重新加载
3. 等待页面完全加载（约3-5秒）
4. 打开个人中心，检查主题按钮是否解锁

**方案B: 检查控制台日志**
1. 按 `F12` 打开开发者工具
2. 切换到 `Console` 标签
3. 查找以下日志：
   ```
   [主题] 开始监听自动登录事件
   [主题] 已注册 userLoggedIn 事件监听器
   自动登录成功: [用户名]
   [事件] 已触发 userLoggedIn 事件
   [主题] 🎉 收到用户登录事件，立即更新按钮
   [主题] 会员类型: SVIP
   [主题] 按钮已解锁
   ```
4. 如果看到"收到用户登录事件"，说明修复成功

**方案C: 手动触发更新**
```javascript
// 在控制台执行
window.themeManager?.updateThemeToggleButton();
```

#### 技术说明
```javascript
// 新增：立即检查机制（防止事件已触发）
listenForAutoLogin() {
    // 注册事件监听器
    window.addEventListener('userLoggedIn', handleUserLogin);
    
    // 立即检查是否已经有用户登录
    if (window.currentUser) {
        console.log('[主题] 检测到用户已登录，立即更新');
        setTimeout(() => this.updateThemeToggleButton(), 100);
    }
    
    // 快速轮询（检测到用户后自动停止）
    const checkUserChange = setInterval(() => {
        if (window.currentUser && !lastUser) {
            this.updateThemeToggleButton();
            clearInterval(checkUserChange); // 停止检查
        }
    }, 500);
}
```

### 问题2: 主题切换后刷新页面又恢复了

#### 症状
- 切换到极简主题成功
- 刷新页面后又变回默认主题

#### 原因
浏览器禁用了本地存储（localStorage）或存储空间已满。

#### 解决方案

**检查本地存储**
1. 按 `F12` 打开开发者工具
2. 切换到 `Application` 或 `存储` 标签
3. 查看 `Local Storage` 是否可用
4. 检查是否有 `exam_theme_preference` 键

**启用本地存储**
1. 浏览器设置 → 隐私和安全
2. 确保"允许网站保存和读取Cookie数据"已启用
3. 刷新页面重试

**清理存储空间**
1. 删除不必要的浏览器数据
2. 确保有足够的存储空间
3. 重新切换主题

### 问题3: 控制台显示"用户未登录"

#### 症状
- 控制台反复显示 `[主题] 用户未登录`
- 但实际上已经登录

#### 原因
- 自动登录延迟
- `window.currentUser` 对象未及时初始化

#### 解决方案

**等待自动修复**
- 主题管理器会自动重试10次（每次间隔1秒）
- 通常在2-10秒内会自动识别登录状态
- 无需手动操作

**手动触发**
1. 打开个人中心
2. 关闭再重新打开
3. 主题按钮会重新检查权限

**检查登录状态**
```javascript
// 在控制台执行以下命令检查
console.log('用户对象:', window.currentUser);
console.log('会员类型:', window.currentUser?.get('membershipType'));
```

### 问题4: VIP会员想使用主题功能

#### 症状
- 是VIP会员
- 主题按钮显示锁定
- 提示需要SVIP/SSSVIP

#### 原因
主题功能仅限SVIP和SSSVIP会员使用。

#### 解决方案

**升级会员**
1. 点击主题按钮
2. 查看会员升级弹窗
3. 选择SVIP或SSSVIP套餐
4. 完成支付即可使用

**会员对比**
| 功能 | VIP | SVIP | SSSVIP |
|------|-----|------|--------|
| 基础刷题 | ✅ | ✅ | ✅ |
| 错题本 | ✅ | ✅ | ✅ |
| 收藏题库 | ✅ | ✅ | ✅ |
| **主题功能** | ❌ | ✅ | ✅ |
| 专属特权 | ❌ | ❌ | ✅ |

### 问题5: 学习统计块在极简主题下消失

#### 症状
- 切换到极简主题后
- 个人中心的4个学习统计块看不见

#### 原因
样式覆盖不完整，统计块的颜色与背景色相同。

#### 解决方案

**已修复**
- 最新版本已修复此问题
- 统计块现在使用深灰色背景 `#1a1a1a`
- 文字使用白色 `#ffffff`
- 边框使用灰色 `#444444`

**如果仍有问题**
1. 清除浏览器缓存
2. 强制刷新页面（Ctrl + F5）
3. 重新切换主题

### 问题6: 主题按钮找不到

#### 症状
- 打开个人中心
- 找不到主题设置按钮

#### 原因
- 按钮位置不明显
- 页面未完全加载

#### 解决方案

**查找位置**
1. 打开个人中心
2. 向下滚动页面
3. 在"数据管理"和"学习统计"之间
4. 查找"主题设置"标题

**按钮特征**
- 标题：🎨 主题设置
- 按钮文字：极简主题 或 默认主题
- 小字提示：黑底白字，无粒子背景

### 问题7: 极简主题太暗，看不清

#### 症状
- 极简主题背景太黑
- 文字对比度不够

#### 原因
这是设计特性，黑底白字适合夜间使用。

#### 解决方案

**切换回默认主题**
1. 打开个人中心
2. 点击"默认主题"按钮
3. 恢复彩色界面

**调整屏幕亮度**
1. 增加屏幕亮度
2. 调整房间光线
3. 使用护眼模式
